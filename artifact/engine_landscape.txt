# Feature Extraction Framework: Level 1 - Position Level (The State)
## Category: Decision Landscape (The Breadth)

**Description:** This category describes the choice architecture of the position. It measures how many "good" options exist and the risk profile of the current state.

### Extracted Features

* **Viable_Move_Count:**
  - Description: The number of legal moves that stay within a competitive threshold (e.g., 0.50) of the top engine move.
  - Data Type: Integer

* **PV1_PV2_Gap:**
  - Description: The evaluation difference between the best move and the second-best move. A large gap signifies a "forced" or "only" move.
  - Data Type: Float

* **Blunder_Risk_Index:**
  - Description: The percentage of legal moves that lead to a catastrophic evaluation drop (>2.00). High values indicate a "minefield."
  - Data Type: Float (0.0 - 1.0)

* **Evaluation_Consistency:**
  - Description: Tracks how often the top-ranked move changes during the engine's search. Low consistency flags "noisy" or "volatile" positions.
  - Data Type: Integer (count of PV changes)

* **Drawishness_Factor:**
  - Description: The count of moves that keep the evaluation within a "dead draw" range (-0.2 to +0.2).
  - Data Type: Integer
