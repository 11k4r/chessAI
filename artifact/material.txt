# Feature Extraction Framework: Level 1 - Position Level (The State)
## Category: Material Balance

**Description:** This category isolates the pure mathematical inventory of the pieces on the board, separate from geographical factors, positional modifiers, or structural nuances (which are handled in categories like Pawn Structure or Coordination).

### Extracted Features

* **Total_Material_Difference:**
  - Description: The raw numerical advantage or deficit. Can be calculated using standard human values (Q=9, R=5, B/N=3, P=1) or engine-specific base values.
  - Data Type: Float / Integer

* **Piece_by_Piece_Delta:**
  - Description: The exact discrepancy in the inventory, highlighting what specific pieces are unmatched on the board (e.g., "White: Rook | Black: Bishop, Knight").
  - Data Type: String / Dictionary of unmatched pieces

* **Bishop_Pair_Status:**
  - Description: Indicates whether either player possesses both of their bishops.
  - Data Type: Categorical (White_Pair, Black_Pair, Neither, Both)

* **Major_Imbalances:**
  - Description: Categorical flags for complex, asymmetrical material trades that fundamentally alter the game's nature.
  - Categories: "Queen vs Two Rooks", "Queen vs Three Minor Pieces", "Exchange Imbalance" (Rook vs Minor Piece + Pawn/s), "None".
  - Data Type: Categorical List

* **Minor_Piece_Ratio:**
  - Description: The specific composition of the minor pieces remaining for both sides.
  - Examples: "2N vs 2B", "B+N vs B+N", "2B vs B+N".
  - Data Type: String

* **Pawn_Count_Differential:**
  - Description: The raw difference in the number of pawns on the board, isolated from major/minor piece counts.
  - Data Type: Integer (e.g., +2 for White)

* **Total_Non_Pawn_Material:**
  - Description: The sum of all active pieces excluding pawns. Used mathematically as the primary trigger to define the current "Game Phase" (Opening, Middlegame, Endgame).
  - Data Type: Integer / Float

* **Sufficient_Mating_Material:**
  - Description: A strict check on whether each side retains the minimum required pieces to legally force a checkmate (e.g., flagging False when a player is reduced to a lone King and Bishop).
  - Data Type: Boolean (per player)
